1. Role-Based Access Control (RBAC)

1.1 Authorization Policy Overhaul

Before: Authorization was completely bypassed in development with RequireAssertion(_ => true), meaning all users could access every endpoint regardless of role.

After: Proper role-based authorization policies implemented:

AdminOrPMAdmin, PMTask approval/rejection

4. Task Lifecycle Re-Engineering

Status Flow

ToDo → InProgress → InReview → Done (Approved)
InReview → Rejected → back to InProgress

Endpoints

EndpointMethodRoleDescription
/api/tasks/{id}/submit-reviewPOSTAssigneeMove InProgress → InReview
/api/tasks/{id}/approvePOSTAdmin, PMMove InReview → Done (records reviewer)
/api/tasks/{id}/rejectPOSTAdmin, PMMove InReview → InProgress
PATCH /api/tasks/{id}/statusPATCHAnyDirect status change (Kanban drag)

Notifications

  • TaskAssigned: Sent to assignees when a task is created or updated with new assignees
  • TaskInReview: Sent to all Admin/PM users when a task is submitted for review
  • TaskApproved: Sent to assignees when a task is approved
  • TaskRejected: Sent to assignees when a task is returned for revision
  • TaskCompleted: Sent to assignees when a task is marked Done

New Fields

  • priority — Low, Medium, High, Urgent (default: Medium)
  • estimated_hours — Effort estimation for capacity planning
  • reviewer_id / reviewed_at — Records who approved/rejected and when

7. Security Audit & Fixes

🔴 CRITICAL: Authorization Bypass FIXED

Issue: RequireAssertion(_ => true) in Program.cs bypassed all authorization — any user could access any endpoint.

Fix: Replaced with RequireAuthenticatedUser() default policy plus named role-based policies. Every mutation endpoint now has explicit [Authorize(Policy = "...")] attributes.

OWASP: A01:2021 – Broken Access Control

Program.cs
🔴 CRITICAL: Unrestricted Role Changes FIXED

Issue: PUT /api/auth/role had no authorization — any user could elevate their own privileges to Admin.

Fix: Added [Authorize(Policy = "AdminOnly")] to the ChangeRole endpoint.

OWASP: A01:2021 – Broken Access Control

AuthController.cs
🔴 HIGH: TimeLogsController Auth Disabled FIXED

Issue: [Authorize] was commented out on the TimeLogsController — unauthenticated users could CRUD time logs.

Fix: Re-enabled [Authorize] attribute.

OWASP: A07:2021 – Identification and Authentication Failures

TimeLogsController.cs
🟡 MEDIUM: CORS Allows Any Origin FIXED

Issue: AllowAnyOrigin() in CORS policy — allowed requests from any domain, creating potential for CSRF-like attacks on browsers.

Fix: Configured WithOrigins() using an AllowedOrigins array in appsettings.json (defaults to localhost:5173 and localhost:8080). Also enabled AllowCredentials().

OWASP: A05:2021 – Security Misconfiguration

Program.cs appsettings.json
🟡 MEDIUM: XSS via Stored Content FIXED

Issue: User-submitted content (messages, notes) was stored and returned without sanitization, allowing stored XSS.

Fix: Created InputSanitizer service with script tag removal and event handler stripping. Applied to message content on creation.

OWASP: A03:2021 – Injection

InputSanitizer.cs MessagesController.cs
🟡 MEDIUM: No Rate Limiting on Auth FIXED

Issue: No rate limiting on login endpoint — vulnerable to brute-force password attacks.

Fix: Implemented RateLimitService (in-memory, 10 attempts per 60-second window per IP). Returns HTTP 429 when exceeded.

OWASP: A07:2021 – Identification and Authentication Failures

RateLimitService.cs AuthController.cs Program.cs
🔵 LOW: JWT Key in appsettings.json NOTED

Issue: JWT signing key is hardcoded in appsettings.json, which could be committed to source control.

Recommendation: Move to environment variables or Azure Key Vault / AWS Secrets Manager for production. Use appsettings.Production.json with secret management.

OWASP: A02:2021 – Cryptographic Failures

🔵 LOW: SQL Injection N/A — Protected

Entity Framework Core uses parameterized queries by default. No raw SQL detected anywhere in the codebase. No action needed.
